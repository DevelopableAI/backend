Generate Zod schemas for this entity: one for CREATE (direct route), one for nested CREATE (if applicable), and one for UPDATE.

Rules:
- For CREATE: required fields are required. Optional fields and fields with defaults are optional.
- For UPDATE: all fields are optional (partial update pattern).
- SERVER-INJECTED fields: if you see a comment like "SERVER-INJECTED (always): fieldName ...", omit that field entirely from ALL schemas — it is injected by the controller and must never be accepted from user input.
- PARENT-FK fields: if you see a comment like "PARENT-FK: fieldName is required in the request body for direct POST ..." then:
  - Include fieldName as required in ${entityNameLower}CreateSchema (direct route needs it from the body)
  - Exclude fieldName from ${entityNameLower}CreateNestedSchema (injected from URL param in nested routes)
- Apply sensible constraints based on field names and types:
  - email → z.string().email({ message: 'Please provide a valid email address' })
      NOTE: do NOT add .min(1) or .nonempty() to email fields — .email() already rejects empty strings
  - password → z.string().min(8).max(128) with complexity note if needed
  - url / website → z.string().url()
  - name / title → z.string().min(1).max(255).trim()
  - username → z.string().min(1).max(100).trim() — do NOT add regex constraints; allow any characters
  - slug → z.string().transform(s => s.toLowerCase().replace(/[^a-z0-9]+/g, '-').replace(/^-|-$/g, '')) — sanitize on input rather than reject
  - description / body / content → z.string().min(1).max(10000)
  - age → z.number().int().min(0).max(150)
  - numeric quantities (price, amount, count) → z.number().min(0)
  - boolean fields → z.boolean()
  - datetime fields → z.string().datetime() or z.coerce.date()
  - enums → ONLY use z.enum([...]) when the field's ts_type is a named TypeScript enum (e.g. "Role", "Status") — never when ts_type is "string". Do NOT infer enum values from field names alone.
  - unique fields → no special schema change needed, the database handles uniqueness
- Use .trim() on string inputs where appropriate.
- Do not add .optional() to the update schema fields individually — wrap the whole object with .partial() instead.

Output only the schema const declarations, nothing else. Use exactly these variable names:
- `${entityNameLower}CreateSchema` — always required
- `${entityNameLower}CreateNestedSchema` — only when PARENT-FK comments are present; must exclude all PARENT-FK fields
- `${entityNameLower}UpdateSchema` — always required
